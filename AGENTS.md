# Documentation Guidelines

## Scope

These rules apply to every memo, note, summary, and research document added to this repository.

## Required workflow

1. Read every existing document in the repo before extending or adding a new one.
2. Identify the document's key factual claims.
3. Verify those claims with external references before finalizing the document.
4. Prefer primary sources over commentary.

## Source preference

Use sources in this order of preference:

1. Peer-reviewed papers and official journal publications.
2. Official white papers, research reports, and technical reports from the originating institution.
3. arXiv or similar preprints when they are the primary public version of the work.
4. Official product or standards documentation when the claim is about a tool, system, or process.
5. Secondary reporting only when primary material is unavailable, and label it clearly as secondary.

## Evidence rules

- Every document with factual claims must include a `## References and Evidence` section.
- Each key fact should have at least one reference. Use multiple references when the claim is important, broad, or likely to be contested.
- Prefer multiple references per key fact when available, especially for workflow guidance, behavioral claims, or research summaries.
- Distinguish direct evidence from interpretation. If a sentence is an inference or synthesis rather than a directly measured result, label it as such.
- Do not treat confident prose as evidence. Unsupported claims should be narrowed, labeled as interpretation, or removed.

## Document structure

Use this structure by default unless the user asks for a different format:

1. Title
2. Main content
3. `## References and Evidence`

Inside `## References and Evidence`:

- Group evidence by claim, not just by source.
- For each claim, include a short statement of what is supported and then the supporting references.
- Use Markdown links for references.

## Writing rules

- Preserve the author's thesis, but tighten factual wording when the evidence is narrower than the original claim.
- Keep factual claims concrete and falsifiable where possible.
- If a historical analogy is used, state whether it is a documented comparison or an interpretive analogy.
- If research is mixed or incomplete, say so directly.
